# max2alamo-2026 v0.9.0

**First pre-1.0 release.** The format library and 3ds Max 2026 plugin both cover the full v1 feature set. Numbered 0.9 to signal *release-candidate quality, awaiting a stabilization window before the v1.0 commitment* — not "early / incomplete".

## What it does

- **`max2alamo.dle`** — a 3ds Max 2026 plugin that exports scenes to the Alamo `.alo` / `.ala` format used by *Star Wars: Empire at War* and *Forces of Corruption*. Drop-in replacement for the legacy Petroglyph Max 9 plugin, written from scratch against the modern Max SDK.
- **`alamo_format`** — a standalone C++17 library that reads and writes the Alamo format with no Max-SDK dependency. Round-trips 4,929 / 4,929 vanilla `.alo` + `.ala` files byte-identical across the full EaW + FoC corpus.
- **`alo_dump`, `alo_roundtrip`** — command-line tools built on top of the library. Useful for inspection, regression testing, and as building blocks for downstream tooling.

## Quick start

1. **Download `max2alamo.dle`** from the assets below.
2. In Windows Explorer: right-click → **Properties** → **Unblock** (Windows marks DLLs downloaded from the internet as untrusted by default).
3. Copy `max2alamo.dle` into a folder Max can scan — recommended: `<your-mod-project>\plugin\`.
4. In 3ds Max 2026: **Customize → Configure System Paths → 3rd Party Plug-Ins → Add**, point at that folder, restart Max.
5. Verify: **File → Export** should now list **"Alamo Object (\*.alo)"**.

Full setup details (alternative install paths, troubleshooting, Max.log diagnostics) live in [`docs/build.md`](https://github.com/DrKnickers/max2alamo-2026/blob/v0.9.0/docs/build.md).

## Highlights since project start

### Format coverage

- Static meshes with per-vertex normals, tangent/binormal frames, and UVs.
- Skinned meshes with multi-bone weighted skinning (top-4 weights, renormalized per vertex).
- 39 stock Petroglyph shader entries with per-shader parameter export. Stub `.fx` files included for in-Max material preview.
- Lights: Omni, Directional, Spotlight + `.Target` sibling-bone convention.
- Hardpoints via the `Alamo_Proxy` helper class (compatible with both modern and legacy Class_IDs, so files authored in the legacy Max 9 plugin open cleanly).
- Animation: bone rotation + translation tracks, packed-pool encoding, visibility tracks (`0x1007`), multi-clip authoring with one `.ala` per clip.
- Shadow-volume meshes: validated for closed-manifold topology at export time; non-fatal warning matches the legacy Petroglyph exporter's behavior.
- Collision meshes: precomputed AABB tree (`0x1200` chunks) emitted at export time, matching vanilla content's structure. Modders who tag a mesh `Alamo_Collision_Enabled = true` but forget to set the `MeshCollision.fx` shader get a non-fatal warning pointing at the fix.
- Billboards: all 8 modes (Parallel, Face, ZAxis View/Light/Wind, Sunlight Glow, Sun) serialized; authoring convention (face normal points along bone-local −Y) documented.

### Authoring quality-of-life

- **Alamo Utility command panel** with four rollouts: Node Export Options, Quick Selection Utility, Animation Settings, plus a Propagate-visibility-to-descendants helper. Lays out the legacy Petroglyph Max 9 UI faithfully against the modern Max 2026 SDK.
- **Multi-clip animation authoring**: name your clips in-Max via the Animation Settings rollout; export emits one `.ala` per clip with filenames matching Mike Lankamp's importer's auto-discovery glob (`<basename>_<clipname>.ala`).
- **Legacy `.max` file compatibility**: opens scenes saved with the legacy Petroglyph Max 9 plugin in modern Max 2026, including auto-translation of legacy clip metadata into the modern authoring convention. Smooths the path for mod teams migrating off Max 9.

### Testing

- 159 Catch2 unit tests / 927 assertions covering the format library.
- ~50 end-to-end MaxScript harness tests driven by `3dsmaxbatch.exe`, each pinning a specific feature with a paired Python verifier.
- Tier 1 validator runs in both strict mode (our authored output) and loose mode (calibrated against the entire vanilla corpus).
- Round-trip oracle: every emitted `.alo` is re-serialized through `alo_roundtrip.exe`; non-zero exit means a format-library writer regression.

## Known limitations

- The `.dle` is built locally only and attached manually to each release — the Max 2026 SDK is non-redistributable, so CI cannot ship the plugin. If you'd rather build from source, see [`docs/build.md`](https://github.com/DrKnickers/max2alamo-2026/blob/v0.9.0/docs/build.md).
- Every chunk the exporter emits is fully specified — the `0x402` mesh-metadata bbox layout (resolved Phase 9.1) and the collision-tree chunks `0x1200`–`0x1203` (resolved + writer added in Phase 9.2) were the last gaps; both shipped in this release. The working spec is in [`docs/format-notes.md`](https://github.com/DrKnickers/max2alamo-2026/blob/v0.9.0/docs/format-notes.md). One known plumbing gap remains in the writer: it emits a fixed `alD3dVertNU2` vertex-format-name string in `0x10002` for every submesh, leaning on the universal 144-byte rev-2 vertex layout (chunk ID `0x10007`) to carry actual per-vertex data regardless of shader. Neither the per-shader `_ALAMO_VERTEX_TYPE` directive (in the `.fx` headers) nor the per-mesh user-prop override is plumbed through. Tier 4 round-trips through AloViewer haven't surfaced a renderer-side issue, but per-shader vertex-format selection is tracked as a post-v0.9 item.
- AloViewer / Mike Lankamp's importer round-trip is exercised manually per major change as Tier 4 smoke; not automated (no headless AloViewer).

## Reporting issues

Use the GitHub issue tracker. Particularly useful in an issue report:

- Your Max version (must be 2026 for the `.dle`).
- The `.export.log` sidecar from a failing export — the walker dumps a per-material diagnostic block plus any warnings (shadow-volume, missing shaders, etc.).
- For round-trip failures: the original `.alo`, plus `alo_dump` output if you can produce it.
